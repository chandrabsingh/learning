Anima Anandkumar - Retrospective Role of Tensors in ML#

Papers#

  • Tensors

    • A. Anandkumar etal, Tensor Decompositions for Learning Latent Variable Models, JLMR 2014

Presentation-Summary#

Introduction#

  • how did the idea came for?

Probabilistic Graphical Models - 2007/08#

  • from signal processing

  • sensor networks, now called as IoT

  • large class of open fields

    • how to do learning in high dimensions

    • statistically and computationally

    • strength of relationships

  • interdisciplinary connections

  • 2011 - started incorporating hidden variables

    • to model such distributions

    • some variables are hidden

    • image has chair and table

      • image is outdoor (hidden variable)

    • how to learn hidden variable

Unsupervised Learning - Topic Models#

  • wide spread applicability

  • text document

    • want to extract where did each word come from(topic)

    • then calculate the proportion of different topics

    • basic clustering cannot be used, as each document can have multiple topics in it

      • clustering requires one topic in a document

    • proportion of each topic

    • simplicity of model comes from exchangability

      • order of words dont matter

        • first draw topics from proportion vector

        • and then draw words from word-topic matrix

  • this is a graphical model and whats hidden are the topics

  • this needs to be learned and is a challenge

Prior Art#

  • standard approach to solve the problem

    • maximum likelihood (EM or variational inference)

    • this is an NP-hard problem

    • there is variance like Expectation Maximization

      • globally optimal solution not guaranted

    • failure in higher dimensions

      • cannot be scaled

      • neither topics nor word-topic matrix

      • not easy to parallelize

Method of Moments#

  • Alternative to Maximum Likelihood

  • record the empirical moments of observed variables and then try to fit them

    • first order is the mean

    • second order is the pairwise correlation

  • first compute and then find a suited distribution

  • if you use two orders only

    • this is a Gaussian approximation

  • higher order moments are required

  • for this Tensors are required

    • when hidden variables are present

    • to get good estimate of model,

    • higher order moments are needed

  • this is even more needed in multivariate higher order moments

Tensor: Extension of Matrix#

  • What tensors are

  • what is the algebra behind it

  • as tensors are generalization of matrices to higher dimensions

    • what property of matrix algebra is retained and what is different

    • tensors are applied in lot of different areas

  • in quantum studies, this is the foundation and in many other areas

Matrix vs Tensor decomposition#

  • tensor algebra is very different from matrix algebra

  • to decompose tensors into rank 1 components

    • can have non-orthogonal components and still have uniqueness

  • uniquely identify decomposition

    • much less stringent conditions compared to matrix decomposition

  • to look at pairwise correlation matrix of topic model

    • with orthogonality, you wnat words to appear uniquely

      • only then can you guarantee identifiability of the underlying model

      • this is not possible

    • with tensors, there is a mild condition for uniqueness

      • topic vector should be linearly independent

      • topics are not representing something redundant in terms of linear independence

      • tensors give lot more information than matrices

        • because they are modeling higher order moments

        • and give deeper relationship between variables

Tensors for Modeling#

  • topic detection in text

  • co-occurence of word

    • look at the average over the entire corpus

      • add correction terms and

      • decompose that in rank-1 components

        • then recover the underlying topic

  • tensor decomposition for topic discovery

Why Not Tensors#

  • why did others not consider tensors

    • they cared about statistical efficiency worse than pairwise correlation

    • for small data, tensors are not suited

      • loose efficiency

    • doing tensor decomposition is NP-hard

      • linear algebra has guaranteed solutions

Why Tensors#

  • Statistical reasons

    • higher order relationships in data can be incorporated

    • hidden topics can be found

  • Computational reasons

    • can be parallelizable like linear algebra

    • faster than other algos for LDA

    • training and inference can be decomposed

    • guaranteed to converge to global optimum under simple assumptions

Trinity of AI/ML#

  • Algorithms, Compute, Data

    • earlier focusing on Algorithms only

  • in todays world,

    • data is multidimensional

    • DL uses tensors to calculate moments but in implicit manner

    • if models can calculate these moments in explicit manner

      • the hidden information is stored there much more efficiently

  • compute side

    • interesting find

Tensor Primitives#

  • Hardware

    • BLAS Level 1 - 1969 - Vector-Vector

    • BLAS Level 2 - 1972 - Matrix-Vector

    • BLAS Level 3 - 1980 - Matrix-Matrix

    • BLAS Level 4 - Now? - Tensor-Tensor

  • Tensors are less expensive to compute in parallelized manner

    • tensoe- much cheaper if you have right primitives

    • design these new primitives in hardware

      • GPU
